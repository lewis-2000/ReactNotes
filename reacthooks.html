Patterns and Hooks

React Patterns and Custom Hooks

These notes cover practical React architecture: component composition, lifted state, basic context, direct DOM access with useRef, and a step-by-step explanation of how custom hooks are created and used.

Main idea

Split UI into small parts, keep state close to where it is needed, and extract repeated behavior into hooks.

Data loading

Fetch in a page or parent component first, then move the logic into a custom hook when reuse appears.

Pattern 1

Component Composition

Instead of one huge component, split responsibilities into smaller focused components.

Bad Example: Everything in One Component

import { useState } from "react";

function TodoApp() {
  const [todos, setTodos] = useState([]);
  const [text, setText] = useState("");

  function addTodo() {
    setTodos([...todos, text]);
    setText("");
  }

  return (
    <div>
      <h1>Todos</h1>

      <input
        value={text}
        onChange={(e) => setText(e.target.value)}
      />

      <button onClick={addTodo}>Add</button>

      <ul>
        {todos.map((todo, index) => (
          <li key={index}>{todo}</li>
        ))}
      </ul>
    </div>
  );
}

export default TodoApp;
Problem:
  • UI logic, list logic, and input logic are mixed together.
  • The component is harder to test, read, and reuse.

Better Example: Split Components

// App.jsx
import { useState } from "react";
import TodoInput from "./TodoInput";
import TodoList from "./TodoList";

function App() {
  const [todos, setTodos] = useState([]);

  function addTodo(text) {
    setTodos([...todos, text]);
  }

  return (
    <div>
      <h1>Todo App</h1>

      <TodoInput onAdd={addTodo} />
      <TodoList todos={todos} />
    </div>
  );
}

export default App;
// TodoInput.jsx
import { useState } from "react";

function TodoInput({ onAdd }) {
  const [text, setText] = useState("");

  function handleAdd() {
    onAdd(text);
    setText("");
  }

  return (
    <div>
      <input
        value={text}
        onChange={(e) => setText(e.target.value)}
      />
      <button onClick={handleAdd}>Add</button>
    </div>
  );
}

export default TodoInput;
// TodoList.jsx
function TodoList({ todos }) {
  return (
    <ul>
      {todos.map((todo, index) => (
        <li key={index}>{todo}</li>
      ))}
    </ul>
  );
}

export default TodoList;
Key lesson: each component should have one clear job.
Pattern 2

Lifting State Up

When two components need the same data, the closest common parent should own that state.

App ├── SearchBar └── ResultsList
// App.jsx
import { useState } from "react";
import SearchBar from "./SearchBar";
import ResultsList from "./ResultsList";

function App() {
  const [query, setQuery] = useState("");

  return (
    <div>
      <SearchBar onSearch={setQuery} />
      <ResultsList query={query} />
    </div>
  );
}

export default App;
// SearchBar.jsx
function SearchBar({ onSearch }) {
  function handleChange(e) {
    onSearch(e.target.value);
  }

  return <input placeholder="Search..." onChange={handleChange} />;
}

export default SearchBar;
// ResultsList.jsx
function ResultsList({ query }) {
  return <p>Searching for: {query}</p>;
}

export default ResultsList;
The rule is simple: the closest common parent owns the shared state.
Pattern 3

useRef Example

useRef stores a value without causing re-renders. A good beginner use case is focusing an input. 

import { useRef } from "react";

function FocusInput() {
  const inputRef = useRef(null);

  function focusInput() {
    inputRef.current.focus();
  }

  return (
    <div>
      <input ref={inputRef} />
      <button onClick={focusInput}>Focus Input</button>
    </div>
  );
}

export default FocusInput;
React normally updates the UI for you, but refs are useful when you need direct access to a DOM element.
Pattern 4

Simple useContext Example

Context helps when passing props through many levels becomes repetitive.

App └── Navbar └── UserMenu
// UserContext.js
import { createContext } from "react";

export const UserContext = createContext(null);
// App.jsx
import { useState } from "react";
import { UserContext } from "./UserContext";
import Navbar from "./Navbar";

function App() {
  const [user] = useState("Lewis");

  return (
    <UserContext.Provider value={user}>
      <Navbar />
    </UserContext.Provider>
  );
}

export default App;
// UserMenu.jsx
import { useContext } from "react";
import { UserContext } from "./UserContext";

function UserMenu() {
  const user = useContext(UserContext);

  return <p>Welcome {user}</p>;
}

export default UserMenu;
Context lets components share data without passing props through every intermediate component.
Pattern 5

Simple folder architecture

Organize by responsibility so the project is easy to scan.

src ├── components │ ├── TodoInput.jsx │ └── TodoList.jsx │ ├── pages │ └── Home.jsx │ ├── context │ └── UserContext.js │ ├── hooks │ └── useFetch.js │ ├── App.jsx └── main.jsx

components

Reusable pieces like buttons, forms, lists, and cards.

pages

Full screens that usually load data and compose components.

hooks

Reusable logic shared by multiple components.

context

Shared state providers for broader app data.

Making Hooks

How custom hooks are made and used

A custom hook is just a JavaScript function that starts with use and calls other hooks inside it.

Typical process: 1. Notice repeated component logic 2. Move that logic into a function named useSomething 3. Keep state and effects inside that function 4. Return the values or functions the UI needs 5. Call the hook from a component 
// hooks/useCounter.js
import { useState } from "react";

function useCounter(initialValue = 0) {
  const [count, setCount] = useState(initialValue);

  function increment() {
    setCount((current) => current + 1);
  }

  function decrement() {
    setCount((current) => current - 1);
  }

  function reset() {
    setCount(initialValue);
  }

  return { count, increment, decrement, reset };
}

export default useCounter;
// components/Counter.jsx
import useCounter from "../hooks/useCounter";

function Counter() {
  const { count, increment, decrement, reset } = useCounter(0);

  return (
    <div>
      <p>{count}</p>
      <button onClick={increment}>+</button>
      <button onClick={decrement}>-</button>
      <button onClick={reset}>Reset</button>
    </div>
  );
}

export default Counter;
The component stays focused on rendering, while the hook owns the reusable behavior.
Examples

Custom hook examples

1. useFetch Hook

This is a practical example because it extracts data-loading logic into one reusable place. 

// hooks/useFetch.js
import { useState, useEffect } from "react";

function useFetch(url) {
  const [data, setData] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch(url)
      .then((response) => response.json())
      .then((result) => {
        setData(result);
        setLoading(false);
      });
  }, [url]);

  return { data, loading };
}

export default useFetch;
// components/UserList.jsx
import useFetch from "../hooks/useFetch";

function UserList() {
  const { data: users, loading } = useFetch(
    "https://jsonplaceholder.typicode.com/users"
  );

  if (loading) {
    return <p>Loading...</p>;
  }

  return (
    <ul>
      {users.map((user) => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}

export default UserList;
This is the clean version of where data is loaded: the fetch logic can live in the hook, and the component consumes the result.

2. useToggle Hook

// hooks/useToggle.js
import { useState } from "react";

function useToggle(initialValue = false) {
  const [value, setValue] = useState(initialValue);

  function toggle() {
    setValue((current) => !current);
  }

  return [value, toggle];
}

export default useToggle;
import useToggle from "../hooks/useToggle";

function ToggleMessage() {
  const [visible, toggle] = useToggle(false);

  return (
    <div>
      <button onClick={toggle}>Toggle</button>
      {visible && <p>Hello React!</p>}
    </div>
  );
}

3. useLocalStorage Hook

// hooks/useLocalStorage.js
import { useState } from "react";

function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const stored = localStorage.getItem(key);
    return stored ? JSON.parse(stored) : initialValue;
  });

  function updateValue(newValue) {
    setValue(newValue);
    localStorage.setItem(key, JSON.stringify(newValue));
  }

  return [value, updateValue];
}

export default useLocalStorage;
import useLocalStorage from "../hooks/useLocalStorage";

function NameInput() {
  const [name, setName] = useLocalStorage("name", "");

  return (
    <div>
      <input value={name} onChange={(e) => setName(e.target.value)} />
      <p>Hello {name}</p>
    </div>
  );
}
Remember

Key hook rules

  • Hooks must start with use.
  • Hooks can use other hooks.
  • Hooks cannot be called inside loops or conditions.
  • Hooks help reuse logic across components instead of copying code.
Strong React code comes from clear component boundaries, stable data ownership, and reusable hooks.
