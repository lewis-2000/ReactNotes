React Foundations and Structure

Main idea

React apps stay readable when file structure and data flow are predictable.

Start

Setting up a React project

Vite is a good beginner setup because it is fast and minimal.

npm create vite@latest my-react-app

Choose React and JavaScript, then run:

cd my-react-app
npm install
npm run dev
The development server shows a local URL. Open that URL in the browser to see the app.
Files

Important React files

Root folder

  • package.json holds scripts and dependencies.
  • index.html is the single HTML entry point.
  • src/ contains almost all React code.
  • node_modules/ is installed code you usually do not edit.

Inside package.json

  • scripts run commands like npm run dev.
  • dependencies are libraries used in the app.
  • devDependencies are tools used during development.
Beginners often try to put React code in index.html. In a Vite app, React code belongs in src.
Structure

How React files are organized

Keep files grouped by what they do, not by random names.

src ├── components │ ├── Navbar.jsx │ ├── UserCard.jsx │ └── SearchBar.jsx │ ├── pages │ ├── Home.jsx │ └── Profile.jsx │ ├── hooks │ └── useFetch.js │ ├── context │ └── UserContext.js │ ├── App.jsx └── main.jsx

main.jsx

Starts React and renders the root App component.

App.jsx

Top-level layout and shared providers usually live here.

pages

Route-level screens that compose multiple smaller components.

components

Reusable UI pieces with one focused job each.

A simple rule: page files assemble the screen, component files handle smaller UI pieces, and hook files hold reusable behavior.
Components

Components and imports

A React component is a JavaScript function that returns JSX.

function Greeting() {
  return <h1>Hello React</h1>;
}

export default Greeting;
import Greeting from "./components/Greeting";

function App() {
  return (
    <div>
      <Greeting />
    </div>
  );
}

export default App;
Do not:
  • Name components with lowercase letters.
  • Put unrelated logic into the same large component.
  • Manipulate the DOM manually for normal UI updates.
Data

Props and basic state

Props send data into a component. State stores data owned by a component. 

function User({ name, age }) {
  return (
    <p>
      My name is {name} and I am {age} years old
    </p>
  );
}

export default User;
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
    </div>
  );
}

export default Counter;
  • Props come from a parent component.
  • State belongs to the component that created it.
  • Updating state causes React to render again.
Flow

Component tree and the order of React components

React apps are built as a component tree. The root component renders first, then it renders child components beneath it.

App ├── Navbar └── Home ├── Profile └── Posts

Tree to code mapping

// main.jsx
import React from "react";
import ReactDOM from "react-dom/client";
import App from "./App";

ReactDOM.createRoot(document.getElementById("root")).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
// App.jsx
function App() {
  return (
    <>
      <Navbar />
      <Home />
    </>
  );
}
// Home.jsx
function Home() {
  return (
    <>
      <Profile />
      <Posts />
    </>
  );
}
Render order idea: main.jsx -> App.jsx -> Navbar + Home -> Profile + Posts
Data usually flows down this tree through props. Events usually flow back up through callbacks.
Loading

Where data is loaded in React

Load data in the component that owns the screen data, or inside a custom hook used by that component.

Common place for fetching

  • A page component like Home.jsx or UsersPage.jsx.
  • A parent container that passes results down as props.
  • A custom hook like useFetch used by that page or container.

Avoid this

  • Fetching the same data in several child components.
  • Putting all fetch logic directly in small presentational components.
  • Spreading state ownership across unrelated files.
import { useEffect, useState } from "react";

function UsersPage() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch("/api/users")
      .then((response) => response.json())
      .then((result) => {
        setUsers(result);
        setLoading(false);
      });
  }, []);

  if (loading) {
    return <p>Loading...</p>;
  }

  return <UserList users={users} />;
}
Start by loading data in the page. If the logic repeats, move that logic into a custom hook.
React code gets easier to maintain when the file layout, component order, and data ownership stay consistent.
