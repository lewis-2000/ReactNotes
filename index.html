React Practice Notes

1. Setting up a React project (Vite)

We use Vite because it is fast and simple.

npm create vite@latest my-react-app

Choose:

  • Framework → React
  • Variant → JavaScript
cd my-react-app
npm install
npm run dev

Open the browser at the local address shown in the terminal.

2. Important files & folders

Root folder

  • package.json
  • node_modules/ (do not touch)
  • index.html
  • src/

package.json

  • scripts – commands like npm run dev
  • dependencies – libraries used by the app
  • devDependencies – development tools
You usually do not edit package.json manually as a beginner.

3. src folder structure

src/
  components/
  pages/
  App.jsx
  main.jsx
  • Component files start with a capital letter
  • Folders use lowercase
  • One component per file

4. React components

A React component is a JavaScript function that returns JSX.

function Greeting() {
  return <h1>Hello React</h1>;
}

export default Greeting;
Do NOT:
  • Manipulate the DOM manually
  • Return multiple elements without a wrapper
  • Name components with lowercase letters

5. Importing and using components

import Greeting from "./components/Greeting";

function App() {
  return (
    <div>
      <Greeting />
    </div>
  );
}

export default App;

export makes code reusable. import brings it into another file.

6. Passing data to components (props)

function User({ name, age }) {
  return (
    <p>
      My name is {name} and I am {age} years old
    </p>
  );
}

export default User;
<User name="Alex" age={25} />
<User name="Sam" age={30} />
  • Props look like HTML attributes
  • Strings use quotes
  • Numbers and expressions use { }

7. Intro to useState

import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
    </div>
  );
}

export default Counter;
  • count – current value
  • setCount – updates the value
  • State change causes re-render

8. Component Tree (How React Apps Are Structured)

React applications are structured as a component tree. Each component can render other components.

Data flows down the tree via props.

Example Component Tree

In this example:

  • App is the root component
  • Navbar and Home are children of App
  • Profile and Posts are children of Home

Tree to Code Mapping

// App.jsx
function App() {
  return (
    <>
      <Navbar />
      <Home />
    </>
  );
}
// Home.jsx
function Home() {
  return (
    <>
      <Profile />
      <Posts />
    </>
  );
}
If you can draw the tree, you can write the React code.

Next Class: React Hooks

In the next session, we focus on React Hooks — the modern way to manage state, side effects, and performance in functional components.

Hooks we will cover

  • useEffect – side effects and lifecycle behavior
  • useMemo – memoizing expensive calculations
  • useCallback – stable function references
  • useRef – accessing DOM elements and persistent values

Hooks allow you to use React features such as state and lifecycle behavior without writing class components. They encourage cleaner logic, better reuse, and more predictable component behavior.

Key focus: useEffect is the most important hook. We will use it for: data fetching, event listeners, and cleanup when a component unmounts.
